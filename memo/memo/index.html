    Often Forgotten But Important

    忘れがちだが大切なこと

    ローカル Git リポジトリのリモート URL 確認方法

    1. git config --get remote.origin.url
    2. git remote -v
    3. git remote show -n origin
    4. git remote show origin
    5. cat .git/config
    6. git config -l

    Environment Management

    以前、様々な競合コマンドが跋扈していて仮想環境を作るのに何を使えばいいか悩ましかった。
    今は venv 一択で良さそう。

    1. Python 仮想環境でありながら公式だから
    2. 公式系関連エコシステムが充実しているから
    3. pip が使えるから

    引用：2021年のPython仮想環境〜いまやvenvを使わない理由はありません〜 - Qiita

    Python3 Symbolic Link

    python コマンドは Python 2.7、python3 コマンドは Python 3.7 を指したりする。（例：Amazon Linux 2）
    管理しにくいのでまず中のコマンドを確認して、下記のように実行して python コマンドのシンボリックリンクを python3 に紐づけるべき。

    [ec2-user@ip-10-0-1-228 work]$ python -V
Python 2.7.18
[ec2-user@ip-10-0-1-228 work]$ python3 -V
Python 3.7.15
[ec2-user@ip-10-0-1-228 work]$ ll /usr/bin/ | grep python
lrwxrwxrwx 1 root root           7 Sep 14 21:00 python -> python2
lrwxrwxrwx 1 root root           9 Sep 14 21:00 python2 -> python2.7
-rwxr-xr-x 1 root root        7048 May 25  2022 python2.7
-rwxr-xr-x 1 root root        1846 May 25  2022 python2.7-config
lrwxrwxrwx 1 root root          16 Sep 14 21:00 python2-config -> python2.7-config
lrwxrwxrwx 1 root root           9 Dec 29 03:57 python3 -> python3.7
-rwxr-xr-x 2 root root        7048 Oct 31 22:47 python3.7
-rwxr-xr-x 2 root root        7048 Oct 31 22:47 python3.7m
lrwxrwxrwx 1 root root          14 Sep 14 21:00 python-config -> python2-config
[ec2-user@ip-10-0-1-228 work]$ sudo rm -rf /usr/bin/python
[ec2-user@ip-10-0-1-228 work]$ sudo ln -s /usr/bin/python3 /usr/bin/python

    From Activation To Deactivation

    venv による仮想環境構成の流れ。

    • .venvという名の仮想環境を作成
    • アクティベート（パスは通ってないので仮想環境内 bin/activate を実行）※ PS1([username@hostname ~]$ 箇所のこと) のすぐ左隣（つまり左端）に仮想環境名が表示される
    • ディアクティベート（ディアクティベートはパスが通っているためコマンド名のみで可）
    • ディアクティベートすると左端の仮想環境名表示が消える
    [ec2-user@ip-10-0-1-228 sphinx-work]$ python -m venv .venv
[ec2-user@ip-10-0-1-228 sphinx-work]$ source .venv/bin/activate
(.venv) [ec2-user@ip-10-0-1-228 sphinx-work]$ deactivate
[ec2-user@ip-10-0-1-228 work]$

    Sphinx 構築

    sudo yum 類コマンドを実行したら出てきた下記の例外エラーが発生。

    [ec2-user@ip-10-0-1-228 sphinx-work]$ sudo yum -y install python-pip
  File "/bin/yum", line 30
    except KeyboardInterrupt, e:
                            ^
SyntaxError: invalid syntax

    同事象参考にする➞except KeyboardInterrupt, e:となった時の対応方法 - Qiita
    シンボリックリンクを python2 に戻したらうまくいったが、そもそも python3 については sudo yum -y install python3-pip が正しい？（そしてまた python3 に設定） venv する前に、python3-pip がないとうまくいかないため、sudo yum -y install python3-pip をしてから venv 仮想環境を作成する。

    プロジェクト作成

    "newDoc" というプロジェクト名でプロジェクトを作成しビルドする。

    mkdocs new newDoc
cd newDoc
mkdocs build
